# V-Corp 推奨事業と 90日実行計画

> ブルーオーシャン分析・スコアカードの結論を踏まえた、CEO としての最終推奨。

---

## 1. 戦略推奨: 「2レイヤー・ポートフォリオ」

単一事業ではなく、**キャッシュエンジン + 本命プロダクト** の 2レイヤー構成を推奨する。

```
┌────────────────────────────────────────────┐
│ Layer A: キャッシュエンジン (Day 1 から稼働)│
│ ─────────────────────────────────────────  │
│ シニア経営者 Notion 代行 (候補 4)          │
│ → 月 1社 × 60万円 = 月商 60万円から開始   │
└────────────────────────────────────────────┘
                    │
                    ↓ 利益を本命に再投資
                    │
┌────────────────────────────────────────────┐
│ Layer B: 本命 SaaS (Day 30 から並走開発)   │
│ ─────────────────────────────────────────  │
│ AI ガバナンス台帳 SaaS (候補 2)            │
│ → 12ヶ月で MRR 100万円を目標              │
└────────────────────────────────────────────┘
```

### なぜ 2レイヤーか

| 理由 | 説明 |
|---|---|
| **キャッシュフロー** | SaaS は MRR 立ち上がりに 6-12ヶ月かかる。その間の生活費・開発費を稼ぐ必要がある。 |
| **顧客理解** | Notion 代行で経営者の生の声を聞けるため、AI ガバナンス SaaS のペルソナ設計に直結する。 |
| **リスク分散** | 1事業が失敗しても、もう 1事業で生き残れる。 |
| **個人事業主向き** | レイヤー A は受託・労働集約、レイヤー B は資産・スケール。両方やる戦略は組織には難しいが個人なら可。 |

### なぜ FinOps (候補9) を本命にしなかったか

- 候補 9 (FinOps) もスコア最高水準だが、AWS の純正機能進化で 2-3年で陳腐化するリスクがある。
- 候補 2 (AI ガバナンス) は規制ドリブンで「やらないと監査が通らない」需要であり、より持続的。
- ただし、候補 2 が 6ヶ月で PMF 確認できなければ、**候補 9 へピボットする** こと。

## 4. リスクと対策

| リスク | 対策 |
|---|---|
| 候補2 の規制動向が遅れる | 候補9 (FinOps) へピボット可能なよう、技術スタックを共通化 (Next.js + Supabase) |
| 候補4 の引き合いが取れない | 商工会議所・地銀・事業承継士へ個別営業強化、note 連載で SEO |
| 個人事業主の信用不足で B2B 契約が結べない | 業務委託契約書テンプレを弁護士監修で準備、賠償責任保険加入 |
| 1人で稼働限界 | Day 60 から VA (バーチャルアシスタント) 1名・週20時間で雇用可能に |
| 健康・属人化 | 全プロセスを Notion + AI に文書化、週 1日完全休養を死守 |

## 7. 「やらないことリスト」 (Don't Do List)

CEO として明確に拒否すべき選択肢。

- ❌ 法人化を急がない (個人事業主の機動性を活かす)
- ❌ 受託コンサルだけで終わらせない (SaaS 化を諦めない)
- ❌ 全方位 AI スタートアップにならない (1業種特化)
- ❌ 大企業向け機能を作らない (中小・中堅に絞る)
- ❌ オフィスを借りない (固定費は最後まで上げない)
- ❌ 営業代行を雇わない (CEO 自身の声と顔で売る)
- ❌ 補助金前提のビジネスにしない (自力で黒字)
- ❌ X バズ狙いの一発芸をしない (積み上げ型コンテンツ)
